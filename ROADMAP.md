# IntentGuard Roadmap

## Phase 3: Ship-Blocking (Days 1-3) ✅

### 3.1 Docker Container Test ✅
- `scripts/test_docker.sh` — builds and tests all verticals
- Tests all endpoints: `/health`, `/v1/models`, `/v1/classify`, `/v1/feedback`, `/metrics`
- Measures startup time (target < 5s) and image size (target < 500MB)
- Dockerfile parameterized with `ARG VERTICAL=finance`

### 3.2 INT8 Quantization ✅
- Added Optimum `ORTQuantizer` as primary quantization method
- Three-method fallback chain: Optimum → Olive → ORT
- `--quantize-method` CLI arg to force specific method
- `--max-size-mb` arg with warning when exceeded
- `scripts/benchmark_quantization.sh` for method comparison

## Phase 4: Product Readiness (Days 4-7) ✅

### 4.1 Shadow Mode ✅
- `mode` query param on `/v1/classify`: `enforce` (default) or `shadow`
- Shadow mode classifies normally but always returns `decision: allow`
- Real classification in `X-Classification-Shadow` header

### 4.2 Feedback Endpoint ✅
- `POST /v1/feedback` records expected vs actual decision
- Logs to structured JSON, optional JSONL file

### 4.3 Prometheus Metrics ✅
- `/metrics` endpoint with counters, histograms, gauges
- Optional `prometheus_client` dependency

### 4.4 Calibration Fix ✅
- All verticals calibrated with ECE < 0.03

## Phase 5: CI/CD and Publishing (Days 8-10) ✅

### 5.1 GitHub Actions CI ✅
- `.github/workflows/ci.yml`: lint + test + Docker build test
- `.github/workflows/release.yml`: tag-triggered matrix build
  - Docker images pushed to GHCR for all verticals
  - PyPI publish via OIDC trusted publishing

### 5.2 HuggingFace Model Publishing ✅

- `model_cards/template.md` — YAML frontmatter + usage examples

### 5.3 PyPI / Package Publishing ✅
- `pyproject.toml` updated with classifiers, keywords, project.urls
- `publish` optional dep group for huggingface-hub
- `scripts/test_pip_install.sh` — clean venv install verification

## Phase 7: Embeddings Router — v2 Architecture (Days 15-20) ✅

### 7.1 Design ✅
- Single container serving multiple verticals
- Stage 1: vertical router (trained N-way classifier)
- Stage 2: per-vertical classifier (ALLOW/DENY/ABSTAIN)

### 7.2 Build Router ✅
- `src/intentguard/router.py` — `VerticalRouter` class
  - Loads router ONNX model + per-vertical classifiers
  - `route(text)`, `classify(text)`, `route_scores(text)` methods
  - `from_config()` factory for JSON-based configuration
- `training/train_router.py` — trains N-way classifier on combined data
- `training/router_config_template.json` — deployment config template
- `Dockerfile.multi` — multi-vertical container with `ROUTER_ENABLED=true`
- Server integration: router mode in `lifespan()` and `classify()`
- Schema additions: `routed_vertical`, `router_scores`, `policy_pack`
- Health endpoint reports `verticals` list in router mode
- Config additions: `ROUTER_ENABLED`, `ROUTER_CONFIG_PATH` env vars

### 7.3 Policy Pack Layer ✅
- `PolicyPack` model in `policy.py`: `allowed_tools`, `guardrails`, `metadata`
- `policy_packs` field in `PolicySpec` keyed by decision
- `get_policy_pack(decision)` method on `Policy`
- `PolicyPackResponse` in schema, attached to classify response
- All three policy JSONs updated with vertical-specific packs
